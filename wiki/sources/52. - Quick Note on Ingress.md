## Summary

This source introduces ingress at a high level and then immediately gives a naming warning that the course treats as very important. Traffic will come into an ingress and then be routed to either the client or server side of the application. But the main caution is about documentation: the course uses the `ingress-nginx` project from the Kubernetes organization, not the separate `nginxinc/kubernetes-ingress` project from the NGINX company. The names are similar, but the projects are different.

## Key Claims

- `Ingress` is the object type the course will use to bring outside traffic into the app.
- The ingress will route traffic to different services inside the cluster.
- There are multiple ingress implementations in Kubernetes.
- The course is specifically using `ingress-nginx`.
- `ingress-nginx` is the community-led project under the Kubernetes organization.
- The course is not using the separate `nginxinc/kubernetes-ingress` project.
- Documentation can become confusing if you accidentally read the wrong project's docs.
